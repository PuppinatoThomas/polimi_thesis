### Data Deduplication of the **Master** Dataset

*(after standardization → outlier detection → imputation)*

Following the same logic as in your examples:

---

## 1️ Deduplication Steps

1. **Exact duplicate removal**

   * Compare every column value across rows.
   * Drop rows that match *exactly* in every field.

2. **Record linkage (near-duplicate) detection**

   * Use fuzzy matching / key blocking on key fields
     (`street`, `city`, `state`, `zip_code`, `price`, `bed`, `bath`).
   * Treat two rows as non-exact duplicates if they describe the *same property* but differ slightly
     (e.g., minor spelling changes, different missing values, or small numeric drift).

---

## 2️ Detected Duplicates

### **Exact duplicates**

Several rows are true one-to-one repeats after imputation.
Examples (representative):

| Example IDs                                                                                         | Reason |
| --------------------------------------------------------------------------------------------------- | ------ |
| `10437` with address “Cedar Dr, 70 … 30041 … 1681 … 2022-02-23` appears more than once.             |        |
| `53138` with “Lincoln Rd, 47 … Maryland 20720 … 59 … 2022-01-04” appears twice.                     |        |
| `34888` with “383 Market St … 77025 … 1696 … 2021-12-16/blank date” (same core data) appears twice. |        |
| `82978` / “Hill Ln, 71 … Temple TX 76502 … 2344/bed-bath match” repeats.                            |        |

These are dropped so that only one canonical record remains.

---

### **Non-exact (fuzzy) duplicates**

Likely the *same property* but with small discrepancies (address fragments, slightly different price or missing/filled fields).
Typical patterns:

| Pair (Row A ↔ Row B)                                                                                         | Clue |
| ------------------------------------------------------------------------------------------------------------ | ---- |
| “Broadway Ln, 272, Fairview, MI 48621” with and without explicit zip or with price as `$98,500` vs `98,500`. |      |
| “Princess Dr, 277, Livermore CA 94550” appears with/without brokered_by and tiny lot-size differences.       |      |
| “Terrace Blvd, 144, Caldwell ID 83605” shows two entries: one with price blank, one with price 58 or 292k.   |      |
| “Field Dr, 63, Sumter SC 29154” duplicates with slightly different price fields.                             |      |
| “Cedar Dr, 70, Cumming GA 30041” multiple entries differing in price/date but otherwise same.                |      |
| “Hill Ln, 71, Temple TX 76502” repeats with different bed/bath counts but same location.                     |      |

For each group we keep **one consolidated record**, e.g., the row with the most complete or most recent `prev_sold_date`.

---
